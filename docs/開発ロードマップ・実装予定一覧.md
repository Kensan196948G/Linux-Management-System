## 🎯 プロジェクト概要

**Linux管理WebUI** は Webmin 互換のサーバ運用管理システムです。root 操作を `sudo allowlist + 承認ワークフロー` で統制する **セキュリティファースト** 設計を採用しています。

| 項目 | 内容 |
|------|------|
| 対象OS | Ubuntu Linux |
| バックエンド | FastAPI (Python 3.11+) |
| フロントエンド | HTML / CSS / Vanilla JS |
| 認証 | JWT (HS256) |
| DB | SQLite（承認ワークフロー用）|
| 実行ユーザー | `svc-adminui` |
| 開発サーバー | `http://localhost:5012` |
| 本番サーバー | `http://localhost:8000`（または 80/443）|

### セキュリティ3原則

1. **Allowlist First** — 明示的に許可されたものだけ実行する
2. **Deny by Default** — 定義されていない操作は全て拒否する
3. **Audit Everything** — 全操作を監査ログに記録する

## 🔒 開発の絶対ルール（AIエージェント必読）

### ❌ 絶対禁止コード

```python
subprocess.run(..., shell=True)   # shell=True 全面禁止
os.system("...")                  # os.system 禁止
eval(user_input)                  # eval 禁止
exec(user_input)                  # exec 禁止
```

### ✅ 正しい実装パターン

```python
# sudo は必ずラッパースクリプト経由
subprocess.run(["sudo", "/usr/local/sbin/adminui-service-restart", "nginx"], check=True)

# allowlist で入力を検証してから実行
ALLOWED_SERVICES = ["nginx", "postgresql", "redis"]
if service_name not in ALLOWED_SERVICES:
    raise PermissionError(f"Service not allowed: {service_name}")

# 全ての入力に validate_input() を適用
FORBIDDEN_CHARS = [";", "|", "&", "$", "(", ")", "`", ">", "<", "*", "?"]
```

### 必須要件まとめ

| ルール | 説明 |
|--------|------|
| `shell=True` 全面禁止 | subprocess はリスト形式で呼ぶこと |
| allowlist first | 許可リストにないものは拒否 |
| sudo wrapper 経由のみ | `wrappers/adminui-*.sh` を経由すること |
| 全入力バリデーション | `backend/core/validation.py` の `validate_input()` を使用 |
| 監査ログ必須 | `backend/core/audit_log.py` の `audit_log.record()` を呼ぶこと |
| 型ヒント必須 | 全関数に型ヒントを付ける |
| docstring 必須 | 全関数・クラスに docstring を付ける |

## 📁 プロジェクト構成

```
backend/
  api/
    routes/           # FastAPI ルーター（機能ごとに分割）
    main.py           # アプリケーションエントリーポイント・router登録
  core/
    auth.py           # JWT認証・RBAC・PERMISSIONS定義
    approval_service.py  # 承認ワークフロー実行エンジン
    sudo_wrapper.py   # sudo呼び出し抽象化レイヤー
    audit_log.py      # 監査ログ記録
    validation.py     # 入力バリデーション（FORBIDDEN_CHARS）
    constants.py      # 許可リスト定数（ALLOWED_SERVICES 等）

wrappers/
  adminui-*.sh        # sudo ラッパースクリプト（allowlist実装）

frontend/
  dev/                # 開発用 HTML ページ（5012ポート）
  prod/               # 本番用 HTML ページ（8000ポート）
  js/                 # JavaScript モジュール
    api.js            # API クライアント（fetchWithAuth等）
    sidebar.js        # サイドバー描画（renderSidebar）
    env-config.js     # 環境設定（isProd判定）
    theme.js          # テーマ管理

    bootstrap-icons/  # Bootstrap Icons
    chart.js/         # Chart.js
    sortablejs/       # SortableJS

tests/
  unit/               # ユニットテスト
  integration/        # 統合テスト（APIエンドポイント）
  security/           # セキュリティテスト（インジェクション・権限）
  e2e/                # E2Eテスト

docs/                 # ドキュメント集
data/                 # SQLite DB ファイル
logs/                 # アプリケーションログ
```

### 重要な実装規約

```javascript
// localStorage キー（必ずこのキーを使うこと）
localStorage.getItem('access_token')

// 環境判定（ポートベース）
// isProd = true  → ポート 8000 / 80 / 443
// isProd = false → ポート 5012（開発）
const isProd = [80, 443, 8000].includes(window.location.port * 1 || 80);

// スクリプト読み込み順序（必ずこの順番）
// 1. theme.js
// 2. env-config.js
// 3. api.js
// 4. sidebar.js
// 5. renderSidebar('pagename') を明示的に呼ぶ

// サイドバー呼び出し（全ページで必須）
renderSidebar('dashboard');  // ページ名を渡すこと
```

## 🤖 他のAIエージェントへの注意事項

### 絶対に守ること

- `shell=True` は絶対に使わない — セキュリティ監査で即失敗
- `sudo` は必ず `wrappers/adminui-*.sh` 経由で呼ぶ
- 新しいページには必ず `renderSidebar('pagename')` を呼ぶ
- `localStorage` は `'access_token'` キーを使う
- CDN は使わない — 全ライブラリは `vendor/` に格納されている

### 開発環境セットアップ

```bash
# 仮想環境の有効化
source venv/bin/activate

# テスト実行（標準）
venv/bin/python -m pytest tests/ -v

# テスト高速実行（pytest.ini の addopts を無視）
venv/bin/python -m pytest tests/ -v --override-ini="addopts="

# Rate limit DB のクリア（テスト前に必要な場合）
python3 -c "
import sqlite3
conn = sqlite3.connect('data/rate_limit.db')
conn.execute('DELETE FROM login_attempts')
conn.commit()
conn.close()
"

# セキュリティチェック
grep -r "shell=True" backend/ && echo "❌ shell=True detected" || echo "✅ No shell=True"
bandit -r backend/ -ll

# フォーマット
black backend/
flake8 backend/
```

4. `backend/core/constants.py` — 許可リスト定数（変更時は人間承認必須）
5. `backend/core/auth.py` — 権限マトリクス（変更時は人間承認必須）

### 承認フロー（危険操作）

ユーザー追加・削除、シャットダウン、sudo 権限変更などの危険な操作は **必ず承認フロー経由**:

```
POST /api/approval/request   → リクエスト作成
POST /api/approval/{id}/approve → Approver/Admin が承認
（承認後に自動実行）
```
